# API Key Vault -- PRD (Product Requirements Document)

> 생성일: 2026-03-20
> 생성 도구: Show Me The PRD

---

## 1. 제품 개요

### 한 줄 요약
내 PC에서 모든 API 키를 암호화해서 안전하게 보관하고, 프로젝트별로 정리해서 필요할 때 바로 꺼내 쓸 수 있는 데스크톱 앱.

### 해결하는 문제
- API 키가 메모장, .env 파일, 브라우저 탭 등 여기저기 흩어져 있어서 찾기 힘듦
- 키를 평문으로 저장하면 유출 위험 (특히 유료 AI 키는 요금 폭탄 가능)
- 프로젝트마다 어떤 키를 쓰는지 기억하기 어려움
- .env 파일을 매번 수동으로 만들어야 하는 번거로움

### 핵심 가치
- **로컬 전용**: 서버에 키를 보내지 않음. 내 PC에만 저장되어 가장 안전
- **암호화 저장**: 마스터 비밀번호로 잠그고, AES-256 암호화로 보관
- **프로젝트별 정리**: 폴더처럼 프로젝트별로 키를 묶어서 관리
- **원클릭 .env**: 프로젝트 선택하면 .env 파일 자동 생성

---

## 2. 사용자

### 주요 사용자
- **누구**: AI 도구를 활용하는 개발자, 바이브코더, 1인 개발자
- **상황**: 여러 AI 서비스(OpenAI, Claude, Gemini 등)와 클라우드 서비스의 API 키를 사용할 때
- **목표**: API 키를 안전하게 보관하면서도 필요할 때 빠르게 꺼내 쓰고 싶음

### 사용자 시나리오
1. 사용자가 새 프로젝트를 시작하면서 API Key Vault를 열고
2. 프로젝트 폴더를 만들어 필요한 키를 등록하면
3. 나중에 Ctrl+K로 키를 찾아 복사하거나, .env 파일을 자동 생성할 수 있다

## 3. 핵심 기능

| # | 기능 | 설명 | 우선순위 | 복잡도 |
|---|------|------|----------|--------|
| 1 | 키 저장/관리 (암호화) | API 키를 AES-256으로 암호화하여 로컬에 안전하게 저장. 추가/수정/삭제 | P1 (MVP) | 간단 |
| 2 | 프로젝트별 그룹 관리 | 키를 프로젝트 폴더로 분류. "내 쇼핑몰", "블로그" 등 | P1 (MVP) | 간단 |
| 3 | 마스터 비밀번호 잠금 | 앱 실행 시 비밀번호 입력 필수. 금고 잠금장치 역할 | P1 (MVP) | 간단 |
| 4 | 빠른 검색 (Ctrl+K) | 키보드 단축키로 즉시 검색. 이름/태그/제공자로 필터링 | P1 (MVP) | 간단 |
| 5 | 태그 시스템 | 키에 #무료 #유료 #테스트 같은 태그 붙이기 | P1 (MVP) | 간단 |
| 6 | 클립보드 자동 삭제 | 키 복사 후 30초 뒤 클립보드에서 자동 삭제 | P1 (MVP) | 간단 |
| 7 | .env 내보내기/가져오기 | 프로젝트의 키를 .env 파일로 자동 생성하거나, 기존 .env에서 키 가져오기 | P2 | 보통 |
| 8 | 키 사용 내역 로그 | 언제 어떤 키를 복사했는지 기록. 보안 감사용 | P2 | 보통 |
| 9 | 암호화 백업/복원 | PC 교체 시 암호화된 파일 하나로 백업/복원 | P2 | 보통 |
| 10 | 키 상태 대시보드 | 첫 화면에서 만료 임박/미사용/활성 키 현황을 신호등으로 표시 | P2 | 보통 |
| 11 | 시스템 트레이 상주 | 앱 닫아도 트레이에 남아서 빠른 접근 + 알림 수신 | P2 | 보통 |
| 12 | 키 유효성 검증 테스트 | 저장된 키가 실제로 작동하는지 테스트 API 호출로 확인 | P3 | 복잡 |
| 13 | 키 만료/사용량 알림 | 키 만료 D-7 알림, 사용량 초과 경고 | P3 | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
앱 실행 -> 마스터 비밀번호 입력 -> 대시보드 -> 프로젝트 선택 -> 키 복사/관리
```

### 상세 흐름
1. **앱 실행**: 시스템 트레이 아이콘 클릭 또는 앱 아이콘 실행
2. **잠금 해제**: 마스터 비밀번호 입력 (틀리면 재시도, 5회 실패 시 대기)
3. **대시보드**: 키 상태 요약 (만료 임박, 최근 사용, 총 키 수)
4. **키 찾기**: Ctrl+K로 검색하거나, 프로젝트/태그로 탐색
5. **키 사용**: 복사 버튼 클릭 → 클립보드에 복사 → 30초 후 자동 삭제
6. **키 등록**: + 버튼 → 이름, 키값, 제공자, 프로젝트, 태그 입력
7. **.env 생성**: 프로젝트 선택 → "Export .env" → 파일 저장 위치 선택

---

## 5. 성공 기준

- [ ] 마스터 비밀번호 없이는 키에 접근할 수 없다
- [ ] 저장된 키가 파일 시스템에서 평문으로 노출되지 않는다
- [ ] 100개 이상의 키를 등록해도 Ctrl+K 검색이 즉시 동작한다
- [ ] .env 파일 생성이 3클릭 이내로 완료된다
- [ ] 앱 크기가 20MB 이하이다 (Tauri 기준)

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 클라우드 동기화 -- 이유: 로컬 전용이 핵심 가치. 서버 비용과 보안 리스크 증가
- 팀/조직 공유 기능 -- 이유: 개인용 도구. 팀 기능은 복잡도가 급격히 증가
- 모바일 앱 -- 이유: 데스크톱 전용. API 키는 주로 PC에서 사용
- 브라우저 확장 -- 이유: 데스크톱 앱으로 충분. 공격 표면 증가 우려
- 자동 키 교체(rotation) -- 이유: 각 서비스마다 API가 달라서 범용 구현 불가
