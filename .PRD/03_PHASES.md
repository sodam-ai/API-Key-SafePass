## Phase 1: MVP

### 목표
앱을 열면 마스터 비밀번호로 잠금을 풀고, API 키를 안전하게 저장/검색/복사할 수 있다.

### 기능
- [ ] 키 저장/관리 (AES-256 암호화)
- [ ] 프로젝트별 그룹 관리
- [ ] 마스터 비밀번호 잠금
- [ ] 빠른 검색 (Ctrl+K)
- [ ] 태그 시스템
- [ ] 클립보드 자동 삭제 (30초)

### 데이터
- AppSetting, Project, ApiKey, Tag, ApiKeyTag

### 인증
- 마스터 비밀번호 (로컬 해시 저장, 로그인 서버 불필요)

### "진짜 제품" 체크리스트
- [ ] 실제 SQLite DB 연결 (목업 데이터 X)
- [ ] 실제 AES-256 암호화 (키값이 평문으로 저장되면 안 됨)
- [ ] 마스터 비밀번호 해시 검증 (하드코딩된 비밀번호 X)
- [ ] Windows에서 .exe 또는 .msi로 설치 가능
- [ ] 앱을 닫았다 다시 열어도 데이터가 유지됨

## Phase 2: 확장

### 전제 조건
- Phase 1이 안정적으로 동작하는 상태

### 목표
.env 파일 자동 생성, 사용 내역 추적, 백업/복원, 대시보드, 트레이 상주로 실용성을 높인다.

### 기능
- [ ] .env 내보내기/가져오기
- [ ] 키 사용 내역 로그
- [ ] 암호화 백업/복원
- [ ] 키 상태 대시보드
- [ ] 시스템 트레이 상주

### 추가 데이터
- EnvMapping, UsageLog, BackupHistory

### 통합 테스트
- Phase 1 기능 (키 CRUD, 검색, 암호화, 잠금)이 여전히 정상 동작하는지 확인

## Phase 3: 고도화

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중

### 목표
외부 API와 연동하여 키가 살아있는지 자동 확인하고, 만료/사용량 알림을 받는다.

### 기능
- [ ] 키 유효성 검증 테스트 (API 호출로 키 상태 확인)
- [ ] 키 만료/사용량 알림 (만료 D-7 알림, 사용량 80% 경고)

### 추가 데이터
- KeyAlert

### 주의사항
- 각 API 제공자(OpenAI, Claude, AWS 등)마다 검증 API가 다르므로, 제공자별 어댑터 패턴 필요
- 사용량 조회 API가 없는 서비스는 알림 불가 (수동 입력 대안 제공)
- API 호출 시 키가 네트워크를 타므로, HTTPS 필수 + 최소한의 요청만
