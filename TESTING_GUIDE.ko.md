# API Key SafePass — 처음 써보는 사람을 위한 테스트 가이드

> 이 문서는 컴퓨터나 코딩을 한 번도 안 해보신 분도 그대로 따라 하면
> "API Key SafePass" 앱이 잘 작동하는지 직접 눈으로 확인할 수 있도록 만든 안내서입니다.
> 어려운 용어는 최대한 쓰지 않았고, 꼭 필요한 용어는 처음 나올 때 쉬운 말로 풀어서 설명했습니다.
>
> **인터넷이 필요 없습니다.** 이 앱은 여러분의 컴퓨터 안에서만 작동하는 프로그램이라,
> 와이파이가 꺼져 있어도 테스트하는 데 전혀 문제없습니다.

## 0. 시작하기 전에 알아둘 것

### 이 앱이 무엇인가요?
"API Key SafePass"는 여러 사이트에서 쓰는 **API 키**(비유하자면, 프로그램들끼리 서로를 알아보는 데 쓰는
일종의 "출입증")나 **아이디/비밀번호**를 안전하게 저장해 두는 개인용 금고 프로그램입니다.
모든 정보는 암호로 잠긴 채 **내 컴퓨터 안에만** 저장되고, 외부 서버로 전송되지 않습니다.

### "터미널"이 뭔가요?
터미널(다른 이름: 명령 프롬프트, cmd, PowerShell)은 마우스로 클릭하는 대신
**글자로 명령을 입력해서** 컴퓨터에게 시키는 화면입니다. 평소에 안 써보셨어도 괜찮습니다.
아래 1번 항목에서 여는 방법부터 정확히 알려드립니다.

### 준비물
- Windows 컴퓨터 (이 가이드는 Windows 기준입니다)
- 이 프로젝트 폴더: `D:\AI_Dev_Work\2026y\26y_03m_22d_API-Key-SafePass`
- 개발에 필요한 프로그램(Node.js, Rust 등)은 이미 설치되어 있다고 가정합니다.
  (이 부분이 안 되어 있다면 개발자에게 문의해 주세요 — 이 가이드는 "테스트 방법"만 다룹니다.)

### 테스트 중 지켜주시면 좋은 것
- 실제로 중요한 비밀번호나 진짜 API 키를 입력하지 마세요. **테스트용 가짜 값**만 사용하세요.
  (예: 비밀번호는 `test1234`, API 키는 `sk-ant-test-1234567890` 같은 식으로 아무 의미 없는 값)
- 각 단계는 순서대로 진행하는 것을 추천합니다. 뒤 단계가 앞 단계 결과를 이용하기 때문입니다.

## 1. 앱 실행하기

### 1-1. 터미널 열기 (파일 탐색기에서 시작)

1. 키보드에서 **윈도우 키 + E**를 누르세요. "파일 탐색기" 창이 열립니다.
2. 왼쪽 목록에서 **"내 PC"** 또는 **"이 PC"**를 클릭하세요.
3. **D 드라이브**를 더블클릭하세요.
4. 순서대로 더블클릭해서 들어가세요:
   `AI_Dev_Work` → `2026y` → `26y_03m_22d_API-Key-SafePass`
5. 지금 보이는 폴더 안에 `src`, `src-tauri`, `package.json` 같은 파일/폴더들이 보이면 제대로 온 것입니다.
6. 이 창의 **맨 위 주소 표시줄** (지금 폴더 경로가 적혀 있는 긴 칸)을 **마우스로 클릭**하세요.
   - 주소 표시줄을 클릭하면 경로 전체가 파란색으로 선택됩니다.
7. 그 상태에서 키보드로 `cmd` 라고 입력하고 **Enter 키**를 누르세요.
8. 검은색(또는 흰색) 화면의 새 창이 뜹니다. 이게 **터미널**입니다.
   창 제목이나 맨 위 경로에 `26y_03m_22d_API-Key-SafePass`가 보이면 정확한 위치에서 열린 것입니다.

> **팁**: 이 터미널 창은 이번 테스트가 끝날 때까지 계속 열어두세요.
> 앱이 실행되는 동안 여기에 여러 상태 메시지가 계속 나타나는데, 이는 정상입니다.

### 1-2. 앱 실행 명령 입력하기

1. 방금 연 터미널 창에 커서를 놓고, 아래 글자를 **그대로** 입력하세요. (복사해서 붙여넣어도 됩니다 —
   터미널에서는 마우스 오른쪽 버튼 클릭으로 붙여넣기가 되는 경우가 많습니다.)

   ```
   npm run tauri dev
   ```

2. **Enter 키**를 누르세요.

3. 화면에 여러 줄의 글자가 스크롤되며 지나갑니다. 이 과정은 처음 실행 시 **1~5분 정도** 걸릴 수 있습니다
   (컴퓨터 부품 준비 과정이라고 생각하시면 됩니다). 화면이 멈춘 것처럼 보여도 기다려 주세요.

4. **정상적으로 잘 되고 있다는 신호**:
   - `VITE ... ready in ...ms` 같은 초록색 글자가 보인다.
   - 잠시 후 `Compiling api-key-safepass ...` 같은 글자가 보인다.
   - 마지막에 **새로운 프로그램 창이 화면에 떠오른다** (제목이 "API Key SafePass"인 창).

5. **문제가 있다는 신호** (이 경우 13번 "문제가 생기면" 섹션 참고):
   - 빨간색 글자로 `error` 라는 단어가 여러 번 보인다.
   - 몇 분이 지나도 새 창이 뜨지 않고 터미널이 멈춰 있다.
   - `command not found` 또는 `찾을 수 없습니다` 같은 메시지가 뜬다.

6. 앱 창이 떴다면 성공입니다! 이제 다음 단계로 넘어가세요.

> **테스트를 마친 후 앱을 끄는 법**: 앱 창의 오른쪽 위 X 버튼을 누르면 창이 닫힙니다.
> 터미널 창도 완전히 종료하려면 터미널 안을 클릭하고 **Ctrl + C**를 누른 뒤,
> `배치 작업을 종료할까요? (Y/N)` 이라고 물으면 `Y`를 입력하고 Enter 하세요.

## 2. 처음 실행 — 마스터 비밀번호 만들기

처음 앱을 실행하면 "마스터 비밀번호"(이 금고 전체를 여는 단 하나의 비밀번호)를 만드는 화면이 나옵니다.

> **⚠ 시작 전 확인 (중요)**: 화면에 "비밀번호를 입력하여 잠금을 해제하세요"라고 나오고 "비밀번호 확인" 칸이
> **없다면**, 이 컴퓨터에 이 앱을 실행한 기록이 이미 있어서 예전에 만든 금고(볼트)가 존재하는 것입니다.
> 이 경우 `test1234`를 입력하면 **"비밀번호가 틀렸습니다"**가 뜨는 게 정상입니다 (버그 아님 — 예전 진짜
> 비밀번호가 아니라서 그런 것뿐입니다).
>
> - 화면에 "비밀번호 확인" 칸이 **있으면** → 정말 처음 실행이니 아래 순서를 그대로 따라 하시면 됩니다.
> - "비밀번호 확인" 칸이 **없으면(=이미 볼트 있음)** → 아래 방법으로 깨끗하게 초기화한 뒤 진행하세요.
>   1. 앱을 완전히 종료하세요.
>   2. **윈도우 키 + R**을 누르고 `%APPDATA%\com.apikeyvault.app`을 입력한 뒤 Enter 하세요.
>   3. 폴더 안의 `vault.db` 파일을 **삭제하지 말고**, 이름만 `vault.db.bak`으로 바꾸세요
>      (마우스 오른쪽 클릭 → 이름 바꾸기). 예전 데이터는 지워지지 않고 그 이름으로 그대로 보존됩니다.
>   4. 1번 항목대로 앱을 다시 실행하면, 아래 설명대로 "비밀번호 확인" 칸이 있는 진짜 첫 화면이 나옵니다.

### 정상 흐름 테스트
1. 비밀번호 입력 칸에 테스트용 비밀번호를 입력하세요. 예: `test1234`
2. 비밀번호 확인 칸에 **똑같이** `test1234`를 입력하세요.
3. "만들기" 또는 이에 해당하는 버튼을 누르세요.
4. **복구 키(Recovery Key)** 화면이 나오는지 확인하세요. 이 화면에는 여러분이 마스터 비밀번호를
   잊어버렸을 때 쓸 수 있는 긴 코드가 나옵니다.
   - 테스트 중이니 실제로 어딘가에 적어둘 필요는 없지만, **화면에 코드가 정상적으로 표시되는지**는 확인해주세요.
5. 이후 안내에 따라 다음 화면(잠금 해제 후 메인 화면)으로 넘어가는지 확인하세요.

### 예외 상황 테스트 (일부러 틀리게 입력해보기)
아래 두 가지는 **일부러 잘못된 값**을 넣어서 앱이 올바르게 알려주는지 확인하는 테스트입니다.
확인 후에는 다시 정상적인 값으로 마스터 비밀번호를 만들어 주세요.

- **너무 짧은 비밀번호**: 비밀번호 칸에 `12` 처럼 6글자 미만을 입력하고 만들기를 눌러보세요.
  → 화면에 "비밀번호가 너무 짧습니다" 같은 **오류 메시지**가 떠야 합니다. (그냥 진행되면 문제입니다.)
- **비밀번호와 확인이 다를 때**: 비밀번호 칸에는 `test1234`, 확인 칸에는 `test9999`를 입력해보세요.
  → 화면에 "비밀번호가 일치하지 않습니다" 같은 **오류 메시지**가 떠야 합니다.

## 3. 잠그고 풀어보기 (그리고 비밀번호 5번 틀렸을 때)

1. 앱 화면 어딘가에 있는 **"잠그기"** 버튼(자물쇠 모양 아이콘일 수 있습니다)을 눌러보세요.
2. 비밀번호 입력 화면으로 돌아가는지 확인하세요.
3. 방금 만든 비밀번호(`test1234`)를 입력해서 다시 들어가지는지 확인하세요.

### 잠금 방지 기능 테스트 (일부러 5번 틀리기)
1. 다시 "잠그기"를 눌러 잠금 화면으로 가세요.
2. 일부러 **틀린 비밀번호**(예: `wrong1`, `wrong2`, `wrong3`, `wrong4`, `wrong5`)를 **5번 연속** 입력해보세요.
3. 5번째 틀린 시도 후, 화면에 **"60초 동안 잠김"** 같은 안내 메시지가 뜨는지 확인하세요.
   이건 다른 사람이 비밀번호를 계속 무작위로 시도하는 걸 막기 위한 보안 기능입니다.
4. 60초 정도 기다린 후, 원래의 올바른 비밀번호(`test1234`)로 다시 시도해서 정상적으로 열리는지 확인하세요.

## 5. API 키 추가하기

1. 방금 만든 "테스트 프로젝트"를 선택한 상태에서 "새 API 키 추가" 버튼을 눌러보세요.
2. 이름 칸에 `테스트 키` 라고 입력하세요.
3. 키 값(Value) 칸에 아래 **가짜 값**을 입력해보세요 (진짜 API 키가 아닙니다, 테스트용입니다):

   ```
   sk-ant-test-1234567890abcdef
   ```

4. 입력 직후 "Provider(제공자)" 항목이 자동으로 **"Anthropic"**으로 표시되는지 확인하세요.
   (이 앱은 키 값 앞부분의 형태를 보고 어느 서비스의 키인지 자동으로 알아맞히는 기능이 있습니다.)
5. 저장 버튼을 눌러 목록에 "테스트 키"가 나타나는지 확인하세요.

## 6. 검색 기능 테스트

> 이 테스트는 이번 작업에서 **고쳐진 검색 버그**가 실제로 잘 고쳐졌는지 확인하는 중요한 항목입니다.
> (이전에는 검색 결과가 있을 때 앱 내부에서 오류가 날 수 있었습니다.)

1. 키보드에서 **Ctrl + K**를 눌러보세요. 검색 상자가 나타나야 합니다.
2. `테스트`라고 입력해보세요.
3. 방금 만든 "테스트 키"가 검색 결과에 나타나는지 확인하세요.
4. **화면에 빨간 오류 문구나 갑자기 하얗게 변하는 화면(먹통)이 없는지** 확인하세요.
   검색어를 입력했는데 아무 반응 없이 멈춰버리면 문제가 있는 것입니다.
5. 검색 상자를 닫고(Esc 키), 이번엔 존재하지 않는 단어 `zzz없는검색어zzz`로 검색해보세요.
   → "결과 없음" 같은 안내가 뜨고, 역시 오류 없이 잘 처리되는지 확인하세요.

## 9. .env 파일로 내보내기 테스트

2. **메모장**을 열어보세요:
   - 윈도우 키를 누르고 `메모장`이라고 입력한 뒤 Enter 하세요.

4. 아래와 비슷한 형태로 글자가 붙여지는지 확인하세요:

   ```
   테스트_키=sk-ant-test-1234567890abcdef
   ```
   (정확한 변수명 형식은 앱 설정에 따라 다를 수 있습니다. 핵심은 `이름=값` 형태로 붙여지는지입니다.)

## 10. 마스터 비밀번호 바꾸기 테스트 — 가장 중요한 테스트

> **왜 이 테스트가 가장 중요한가요?**
> 마스터 비밀번호를 바꾸면, 앱은 저장된 **모든 정보를 새 비밀번호에 맞춰 다시 암호화**합니다.
> 이 과정에서 8번에서 저장한 "계정 정보(아이디/비밀번호)"까지 포함해서 전부 안전하게 옮겨져야 합니다.
> 이 단계 하나만 실수가 생겨도 데이터가 깨질 수 있는 만큼, **꼭 직접 눈으로 확인해 주세요.**

1. 설정(톱니바퀴 아이콘) 화면으로 들어가서 "마스터 비밀번호 변경" 항목을 찾으세요.
2. 현재 비밀번호(`test1234`)를 입력하세요.
3. 새 비밀번호를 `newTest5678`로 입력하고, 확인 칸에도 똑같이 입력하세요.
4. 변경 버튼을 누르고, "변경 완료" 같은 성공 메시지가 뜨는지 확인하세요.
5. 앱을 잠그세요 (3번 항목에서 했던 "잠그기" 버튼).
6. 이번엔 **옛날 비밀번호(`test1234`)로 잠금 해제를 시도**해보세요.
   → **반드시 실패해야 정상**입니다. (옛날 비밀번호로 열리면 오히려 문제가 있는 것입니다!)
7. 이번엔 **새 비밀번호(`newTest5678`)로 잠금 해제**를 시도하세요.
   → 정상적으로 열려야 합니다.
8. 열린 후, 8번 단계에서 만들었던 "테스트 키"를 다시 열어보세요.
9. "추가 계정 정보" 부분을 펼쳐서, 아이디(`test_user`)와 비밀번호(`Test!Pass1234`)가
   **여전히 정확하게 읽히는지** 확인하세요. 글자가 깨져 보이거나, 빈칸으로 보이거나,
   오류가 뜨면 문제가 있는 것이니 13번 섹션을 참고해 알려주세요.
10. 테스트가 끝나면, 다시 설정에서 마스터 비밀번호를 원래 값(`test1234`)으로 되돌려도 되고,
    그대로 두어도 상관없습니다 (어차피 테스트용 데이터입니다).

   확인할 수 있습니다).
5. 기다린 후 메모장에서 다시 Ctrl + V를 눌러보세요.
   → 아무것도 붙여지지 않거나 예전과 다른(비어있는) 내용이면 정상 작동입니다.
   → 방금 복사했던 키 값이 그대로 다시 붙여지면 자동 삭제가 안 된 것이니 문제로 기록해주세요.

## 13. 문제가 생기면

테스트 도중 예상과 다르게 동작하거나 오류 화면이 보이면, 당황하지 말고 아래 순서로 알려주세요.

1. **터미널 창 전체를 확인**하세요. 빨간 글자나 `error`, `panic`, `Uncaught` 같은 단어가 있는지 살펴보세요.
2. 해당 부분을 **마우스로 드래그해서 선택**한 뒤 **Ctrl + C**(복사)를 누르세요.
   (터미널 안에서의 복사는 마우스 오른쪽 클릭 → "복사"로도 가능한 경우가 많습니다.)
3. 그 내용을 그대로 붙여넣어서 알려주시면, 정확한 원인을 찾는 데 큰 도움이 됩니다.
   **일부만 요약하지 말고, 보이는 오류 글자 전체를 그대로** 붙여주시는 게 가장 정확합니다.
4. 어느 단계(위 목차의 몇 번)에서 문제가 생겼는지도 함께 알려주시면 더 빠르게 확인할 수 있습니다.
5. 화면이 이상하게 보이는 경우(글자가 깨지거나 버튼이 안 눌리는 등)라면,
   가능하면 **화면을 캡처(윈도우 키 + Shift + S)**해서 함께 보여주시면 좋습니다.

> 이 가이드에서 사용한 모든 값(`test1234`, `sk-ant-test-...` 등)은 전부 **테스트용 가짜 값**입니다.
> 실제 서비스에 로그인되는 값이 아니니, 문제 상황을 공유하실 때 이 값들이 그대로 보여도 걱정하지 않으셔도 됩니다.
> 다만 혹시 테스트 중 실수로 **진짜** 비밀번호나 API 키를 입력하셨다면, 그 값만은 가려서 공유해주세요.

## 14. 계정 정보 복사 버튼 테스트 — 새 기능

8번에서 만든 "테스트 키" 안의 아이디·비밀번호를 마우스로 드래그하지 않고, 버튼 클릭 한 번으로 복사할 수 있는 기능입니다.

## 15. .env 파일 가져오기 테스트 — 새 기능

다른 프로그램에서 이미 쓰고 있던 `.env` 파일을, API Key SafePass 안으로 그대로 가져오는 기능입니다.

### 15-1. 테스트용 .env 파일 직접 만들기

1. 윈도우 키를 누르고 `메모장`을 입력해 실행하세요.
2. 아래 두 줄을 그대로 입력하세요 (따옴표도 그대로 입력하는 것이 중요합니다 — 따옴표가 자동으로 제거되는지 확인하는 테스트입니다):

   ```
   TEST_IMPORT_KEY="sk-import-test-123"
   TEST_IMPORT_KEY2=sk-import-test-456
   ```

3. 메모장 위쪽 메뉴에서 **파일 → 다른 이름으로 저장**을 클릭하세요.
4. 저장 창이 뜨면:
   - 왼쪽 목록에서 **"바탕 화면"**을 선택하세요 (찾기 쉬운 위치입니다).
   - 아래쪽 **"파일 이름"** 칸에 `my_test_import.env`라고 정확히 입력하세요 (`.env`까지 꼭 포함).
   - **"파일 형식"** 칸이 있다면 반드시 **"모든 파일 (*.*)"**로 바꿔주세요. 이 설정을 안 바꾸면 메모장이 뒤에 몰래 `.txt`를 붙여서 `my_test_import.env.txt`라는 다른 파일이 되어버립니다.
5. **저장** 버튼을 누르세요.

### 15-2. 가져오기 실행

1. API Key SafePass에서 "테스트 프로젝트"를 열어둔 상태에서, 화면 위쪽의 **"가져오기"** 버튼(아래쪽 화살표 아이콘)을 클릭하세요.
2. 파일 선택 창이 뜨면 **바탕 화면**에서 방금 만든 `my_test_import.env`를 선택하세요.
3. **"2개의 키를 가져왔습니다"** 같은 안내 메시지가 뜨는지 확인하세요.
4. 목록에 `TEST_IMPORT_KEY`, `TEST_IMPORT_KEY2` 두 개의 새 키가 나타나는지 확인하세요.
5. `TEST_IMPORT_KEY`를 복사해서 메모장에 붙여보세요.
   → `sk-import-test-123`처럼 **큰따옴표 없이** 값만 깔끔하게 붙여지면 정상입니다. 따옴표(`"`)까지 같이 붙여지면 문제가 있는 것입니다.

## 16. 복구키 재발급 테스트 — 새 기능

마스터 비밀번호는 그대로 두고, 복구키만 새로 발급받는 기능입니다. (기존 복구키를 잃어버렸거나 다른 사람이 봤을까 걱정될 때 사용합니다.)

4. "확인했습니다" 버튼을 눌러 닫으세요.

## 17. 백업 · 복원 테스트 — 새 기능, 가장 신중하게

> **⚠ 시작 전 확인 (중요)**: 복원 기능은 지금 저장된 데이터 전체를 다른 파일 내용으로 바꾸는 기능입니다.
> 반드시 이 가이드에서 만든 **테스트용 데이터**로만 진행하시고, 실제로 중요한 키가 저장된 상태에서는
> 이 테스트를 하지 마세요. (안전장치가 있어 기존 데이터가 사라지지는 않지만, 신중하게 접근하는 습관이 중요합니다.)

### 17-1. 백업 만들기

1. 설정 화면에서 **"백업 · 복원"** 항목을 찾아, **"지금 백업"** 버튼을 클릭하세요.
2. 저장 위치를 물어보면 **"바탕 화면"**을 선택하고, 파일 이름은 기본값(`api-key-safepass-backup-날짜.db`)을 그대로 두고 저장하세요.
3. **"백업 파일이 저장되었습니다"** 메시지를 확인하세요.
4. 바탕 화면을 직접 확인해서, 방금 말한 이름의 파일이 실제로 생겼는지 눈으로 확인하세요.

### 17-2. 복원 준비하기

1. 다시 설정 화면에서 **"백업에서 복원"** 버튼을 클릭하세요.
2. **"백업 파일(.db) 선택..."**을 클릭하고, 방금 바탕 화면에 만든 백업 파일을 선택하세요.
3. 현재 마스터 비밀번호를 입력하세요.
4. **"복원 준비"** 버튼을 클릭하세요.
5. **"복원 준비 완료"** 화면이 뜨는지, 그리고 "앱을 완전히 종료했다가 다시 실행하면 적용됩니다"라는 안내 문구가 보이는지 확인하세요.
6. "확인했습니다"를 눌러 닫으세요.

### 17-3. 취소도 되는지 확인 (마음이 바뀌었을 때를 위한 안전장치)

1. 설정 화면을 다시 열어보세요. 노란색 배경의 **"복원 대기 중 — 다음 재시작 시 적용됩니다"** 안내가 보이는지 확인하세요.
2. 옆의 **"취소"** 버튼을 눌러보세요.
3. 안내 배너가 사라지는지 확인하세요. (이렇게 하면 예약된 복원이 취소되어, 다음에 앱을 켜도 지금 데이터가 그대로 유지됩니다.)

### 17-4. (선택) 실제 재시작까지 확인하기

이 단계는 필수는 아니지만, 끝까지 확인하고 싶으신 분만 진행하세요.

1. 17-2번을 다시 진행해 복원을 예약하세요 (이번엔 취소하지 않습니다).
2. 앱 창을 완전히 닫고, 터미널에서도 **Ctrl + C** → `Y` → Enter로 완전히 종료하세요.
3. 1번 항목대로 `npm run tauri dev`로 다시 실행하세요.
4. 잠금 해제 화면이 정상적으로 나오는지, 비밀번호로 잠금을 풀 수 있는지 확인하세요.
5. "테스트 프로젝트"의 키들이 이전과 똑같이 남아있는지 확인하세요.
   (지금 만든 백업은 방금 전 상태를 그대로 담고 있으므로, 복원해도 데이터가 똑같은 것이 정상입니다 — 데이터가 사라지거나 달라지면 문제가 있는 것입니다.)
